Your CommunityPilot field guide.

Find every available slash command, see who can use it, and follow practical examples for setting up your streamer community.

Quick start for server managers

  1. Run /setup and choose your Twitch channel and live-alert destination.
  2. Run /testalert to confirm the bot can send messages and embeds to the selected channel.
  3. Run /settings and /alerts status to review the saved configuration.
  4. Optional: use /alertfilter role to restrict who may be announced, then /alertroute set to give matching roles their own destination channel and ping.

Syntax key: <required> means an option is required; [optional] means it may be omitted. Discord will present each option as a field when you select the slash command.

Commands for server managers

These commands require Discord’s Manage Server permission. Their responses are generally visible only to the manager who runs them, while alerts and previews appear in configured channels.

/setup

Manage Server

Starts the guided setup for the primary Twitch username and alert channel. Run this first in a new server.

/setup

/settings

Manage Server

Displays tracked Twitch channels, the default alert channel, the required alert role, and configured role-based routes.

/settings

/alerts

Manage Server

Customizes the appearance, targeting, and lifecycle of Twitch live alerts.

  • /alerts customize [message] [color] [role] Set a message template, a hex embed color such as #9146FF, and an optional role mention. Message templates can use {streamer} and {url}.
  • /alerts filters [games] [title_keywords] Allow alerts only for comma-separated game names and/or stream titles containing at least one comma-separated keyword.
  • /alerts delivery [stream_end_action] [end_message] [auto_crosspost] Keep, edit, or delete the original alert when a stream ends, or send a separate end message. Auto-crosspost publishes alerts in Discord announcement channels.
  • /alerts status Review the current message, color, role, filters, stream-end action, and crosspost setting.
  • /alerts reset Restore the default alert customization, filters, and delivery settings.

/streamer

Manage Server

Manages the individual Twitch channels monitored by this server.

  • /streamer add <username> Add a Twitch username to automatic monitoring. Enter only the username; an initial @ is accepted and removed.
  • /streamer remove <username> Stop manually tracking the named Twitch channel.
  • /streamer list List the Twitch channels currently tracked by the server.
  • /streamer link <username> <member> Connect a tracked Twitch username to its Discord member so Live Role automation can identify the correct person.
  • /streamer unlink <username> Remove the Discord-member association without necessarily removing the Twitch channel from monitoring.
  • /streamer offline-action <username> <action> Set an individual stream-end action for this creator. Choose the server default, keep the alert, edit it, delete it, or send a separate stream-ended message. The individual setting overrides /alerts delivery only for that streamer.

/alertfilter

Manage Server

Restricts live alerts to linked Discord members who have one required role. This filter fails closed: an unlinked member, a missing role, or a role mismatch prevents the alert.

  • /alertfilter role <role> Require linked streamers to have the selected Discord role before any live alert can be sent.
  • /alertfilter status Show the role currently required for live alerts.
  • /alertfilter clear Remove the eligibility restriction and resume normal alerts.

/alertroute

Manage Server

Sends alerts to different channels and ping roles according to the linked streamer’s Discord roles. If several routes match, the member’s highest matching Discord role takes priority.

  • /alertroute set <streamer_role> <channel> [ping_role] Create or update a route. The streamer role selects the route; the optional ping role is the audience mentioned by the alert.
  • /alertroute remove <streamer_role> Delete the route associated with the selected streamer role.
  • /alertroute list List every role-to-channel route and its configured ping.

/streamteam

Manage Server

Configures automatic shoutouts for an official Twitch Team or individually selected creators.

  • /streamteam setup [team_url] [role] [channel] Save an official Twitch Team URL or team name, an optional role, and an optional text or announcement channel for shoutouts.
  • /streamteam add-member <username> [member] Add an individual team creator and optionally associate a Discord member.
  • /streamteam remove-member <username> Remove an individually added team creator.
  • /streamteam status Review the official team, role, shoutout channel, and tracked membership.
  • /streamteam test Send a safe Stream Team shoutout preview to verify the design and destination.
  • /streamteam disable-team Stop official Twitch Team syncing while preserving separately configured members where applicable.

/liverole

Manage Server

Controls the Discord role assigned to linked members while their Twitch channels are live.

  • /liverole setup <role> Select the Live Role. The Venture CommunityPilot bot role must be above the selected role and have Manage Roles.
  • /liverole status Show the currently configured Live Role.
  • /liverole disable Stop automatic Live Role assignment.

/ticket

Manage Server

Configures a private support-ticket system with an Open Ticket panel and staff controls for claiming, closing, reopening, and deleting tickets.

  • /ticket setup <panel_channel> <category> <staff_role> Post the ticket panel and choose where private ticket channels are created and which staff role can access them.
  • /ticket status Display the current panel channel, ticket category, staff role, system status, and number of open tickets.
  • /ticket disable Stop members from opening new tickets without deleting any existing ticket channels.

/testalert

Manage Server

Sends a preview of the automatic live announcement to the configured alert channel.

/testalert

/testend

Manage Server

Tests the configured stream-end behavior using a temporary alert, without requiring a real Twitch stream to end.

/testend

Troubleshooting & contact

Alerts are not appearing

Run /settings, then /testalert. Confirm the bot can view the alert channel, send messages, and embed links. Review content filters with /alerts status, the eligibility role with /alertfilter status, and role destinations with /alertroute list. Streamers must be linked with /streamer link when role filtering or routing is enabled.

Live Role is not changing

Run /liverole status and confirm the Twitch username is linked with /streamer link. Place the bot’s role above the Live Role in Discord’s role hierarchy.

A command is hidden

Administrative commands appear only to members with Manage Server permission. Ask a server owner or administrator for assistance.
